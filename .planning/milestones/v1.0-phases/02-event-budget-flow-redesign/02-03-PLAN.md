<task type="auto">
  <name>Task 1: Fix participant removal and budget redistribution (UAT Test 7)</name>
  <files>workday-application/src/main/react/features/event/EventBudgetManagementDialog.tsx</files>
  <action>
Merge the two competing useEffects (lines ~74 and ~140) into a single useEffect that handles BOTH participant sync AND budget recalculation in one atomic setMoneyParticipants call.

**Root cause (from debug session):** When participantIds changes, both useEffects fire in the same React commit phase. Effect 1 (line ~74) correctly filters removed participants but does not redistribute. Effect 2 (line ~140) reads stale moneyParticipants from the closure and overwrites Effect 1's correctly-filtered result, causing removed persons to reappear.

**The fix — merge into a single useEffect with dependencies `[participantIds, persons, totalBudget]`:**

1. Delete the existing useEffect at line ~74 (participant sync) AND the existing useEffect at line ~140 (budget recalculation). These are the two effects that call `setMoneyParticipants`.

2. Replace with ONE useEffect that does:
   a. Filter: keep only participants whose personId is in `participantIds` (removes deleted persons)
   b. Add: for each personId in `participantIds` not in current moneyParticipants, create a new entry with default share
   c. Redistribute: calculate `defaultMoneyShare = Math.floor((totalBudget / participantIds.length) * 100) / 100` and apply to all participants NOT in `dirtyMoney` set
   d. Preserve: participants in `dirtyMoney` keep their manually-edited amounts
   e. Call `setMoneyParticipants(result)` ONCE

3. Use a functional update (`setMoneyParticipants(prev => ...)`) to avoid stale closure issues. Read the previous moneyParticipants from the functional updater's `prev` argument, not from the outer closure.

4. Similarly update the dirty set cleanup to happen inside the same effect (remove ids not in participantIds).

5. Keep the existing time participant useEffect (line ~74's time portion) as a separate effect since it does not have the same competing-write problem — but also convert it to use functional update pattern for safety.

**IMPORTANT:** The dependency array for the merged effect should be `[participantIds, persons, totalBudget]`. Do NOT include `moneyParticipants` in the dependency array (that would cause infinite loops). The functional updater pattern (`setMoneyParticipants(prev => ...)`) gives access to current state without needing it in deps.

**Also fix the time participant sync:** The existing useEffect that handles time participants (the first part of the line ~74 effect) should also filter out removed participants. Currently it maps over participantIds correctly but the dirty cleanup should also happen here.
  </action>
