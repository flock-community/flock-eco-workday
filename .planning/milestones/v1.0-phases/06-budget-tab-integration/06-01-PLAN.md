Purpose: This endpoint powers the summary cards in the Budget Allocation tab (TAB-01) and supports both admin and non-admin access patterns (TAB-05). It is also designed to be reusable for the v2 dashboard.
Output: Working GET /api/budget-summary endpoint with wirespec contract, service, controller handler, and integration tests.
</objective>

       ```wirespec
       endpoint BudgetSummary GET /api/budget-summary ? { personId: String?, year: Integer32? } -> {
         200 -> BudgetSummaryResponse
       }
       type BudgetSummaryResponse {
         hackHours: BudgetItem,
         studyHours: BudgetItem,
         studyMoney: BudgetItem
       }
       type BudgetItem {
         budget: Number,
         used: Number,
         available: Number
       }
       ```

    3. Run `npm run generate` from root to generate TypeScript types.
    4. Create BudgetSummaryService.kt in the budget package:

       - Method `getSummary(personUuid: UUID, year: Int): BudgetSummaryResponse`
       - Find active contracts for person using `contractService.findAllActiveByPerson(LocalDate.of(year, 1, 1), LocalDate.of(year, 12, 31), personUuid)` -- filter to ContractInternal instances
       - Sum hackHours, studyHours, studyMoney from all active ContractInternal contracts for that year (a person may have multiple contracts in a year)
       - Get allocations via `budgetAllocationService.findAllByPersonUuid(personUuid, year)` (the domain service, not persistence adapter directly)
       - Sum used amounts: hackTime dailyAllocations hours, studyTime dailyAllocations hours, studyMoney amounts
       - Return BudgetSummaryResponse with budget, used, available (budget - used) for each type
       - Handle edge case: no contracts found -> budget=0 for all types
       - Register as @Service or add @Bean in BudgetAllocationConfiguration
    5. Write BudgetSummaryControllerTest.kt following the existing BudgetAllocationControllerTest pattern:
       - @SpringBootTest + @AutoConfigureMockMvc with admin/non-admin test users
       - Seed test ContractInternal with hackHours=100, studyHours=80, studyMoney=2500.00
       - Seed test allocations (hack time with 8 hours, study money with 500.00)
       - Assert GET /api/budget-summary returns correct budget/used/available values
       - Assert non-admin auto-scoping works
