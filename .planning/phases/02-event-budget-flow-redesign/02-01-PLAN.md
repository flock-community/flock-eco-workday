<objective>
Lift Formik state from EventForm to EventDialog so that budget management sections reactively consume form values (costs, defaultTimeAllocationType, personIds, days) as a single source of truth. Consolidate to a single save button, add per-participant dirty tracking for "update if untouched" behavior, and add unsaved changes warning on dialog close.

Purpose: EVT-05 requires that event form field changes live-update budget sections. Currently EventForm owns Formik internally and budget sections read from disconnected mock data, creating a split-brain problem.

Output: EventDialog owns Formik, EventFormFields accepts formik props, EventBudgetManagementSection derives all defaults from form values, single save button, unsaved changes confirmation.
</objective>
